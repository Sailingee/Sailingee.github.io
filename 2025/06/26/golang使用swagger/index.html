Swagger-go

1.1 swaggo/swag

目前 Go 生态中最流行的 Swagger 文档生成工具是 swaggo/swag，它可以将代码中的注释转换为 Swagger 文档。

1.4 在路由中添加注释

  • @Summary: API 的简短描述
  • @Description: 详细描述
  • @Tags: API 所属的标签
  • @Accept: 接受的请求体类型
  • @Produce: 返回的数据类型
  • @Param: 参数说明
    • 格式：参数名 参数位置 数据类型 是否必填 描述
    • 参数位置：query, path, body, header, formData
  • @Success: 成功响应
  • @Failure: 失败响应
  • @Router: API 路由

1.5 查看文档

启动服务后，访问 http://localhost:8080/swagger/index.html 即可查看生成的 Swagger UI。

2.1 导入到ApiFox

启动你的 Go 服务，确保 http://your-server/swagger/doc.json 或 http://your-server/swagger/doc.yaml 可以访问。

  • 打开 Apifox，进入项目
  • 点击 “项目设置” → “导入数据”
  • 选择 “URL 导入”，输入 Swagger 文档的 URL（如 http://localhost:8080/swagger/doc.json）
  • 点击 “导入”，Apifox 会自动解析并生成 API 文档
